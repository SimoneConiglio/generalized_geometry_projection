Additive Manufacturing Constraints¶

The ggp framework extends the standard Generalized Geometry Projection (GGP) method to support Additive Layer Manufacturing (ALM) constraints. This ensures that the generated topology is printable without internal support structures.

This implementation follows the AMNA (Adapted Moving Node Approach) formulation from:

Bhat, K.V., Capasso, G., Coniglio, S., Morlier, J., Gogu, C. On some applications of Generalized Geometric Projection to optimal 3D printing. Computers & Graphics (2021).

Layer-by-Layer Parameterization¶

In standard GGP, components can change their orientation (\(\theta\)) and vertical position (\(y_c\)) freely. In the ALM formulation (provided by GGP2DALMMapper), components are restricted to horizontal tracks of a fixed height (matching the layer height).

Each component \(j\) in layer \(i\) is defined by only three variables: \(x_{i,j} = [X_{c,i,j}, w_{i,j}, M_{i,j}]\)

where:

  • \(X_{c}\) is the horizontal center.

  • \(w\) is the track width (length of the segment).

  • \(M\) is the track mass variable.

AMNA Box Formulation (Paper Eq. 40-41)¶

The characteristic function for each layer segment uses a product of five regularized Heaviside functions, one for each boundary of the rectangular layer primitive:

\[\rho^{el}_i = \prod_{k=1}^{5} W(\zeta_k)\]

where the \(\zeta\) variables represent signed distances from a Gauss point to each boundary edge:

\[\zeta_1 = -L/2 - x_{loc}, \quad \zeta_2 = x_{loc} - L/2, \quad \zeta_3 = y_{loc} - h/2, \quad \zeta_4 = -h/2 - y_{loc}, \quad \zeta_5 = y_{loc} - h/2\]

and the smoothed Heaviside function \(W(\zeta)\) is the 5th-order polynomial:

\[W(\zeta) = \frac{1}{2} - \frac{15}{16\sigma}\zeta + \frac{5}{8\sigma^3}\zeta^3 - \frac{3}{16\sigma^5}\zeta^5\]

This produces a flat box/block density field for each layer, rather than the rounded bar shape used in the Free GGP formulation (GP method). The transition width \(\sigma\) controls the sharpness of the boundary.

Overhang Angle Constraints¶

To be printable without supports, each layer must be sufficiently supported by the layer beneath it. For a layer height \(\Delta h\) and a maximum allowable overhang angle \(\alpha\) (typically 45°), the following linear constraints are enforced between layer \(i\) and layer \(i+1\) (Paper Eq. 42-45):

\[ \begin{align}\begin{aligned}(x_{c, i+1} + \frac{w_{i+1}}{2}) - (x_{c, i} + \frac{w_i}{2}) \le \Delta h \cdot \tan(\alpha)\\(x_{c, i} - \frac{w_i}{2}) - (x_{c, i+1} - \frac{w_{i+1}}{2}) \le \Delta h \cdot \tan(\alpha)\end{aligned}\end{align} \]

Implementation in GEMSEO¶

These constraints are linear with respect to the design variables. In the provided examples, they are wrapped in an ALMConstraintsDiscipline and passed to the GEMSEO optimizer (MMA), which handles them alongside the volume and compliance objectives.

See the alm_example for a complete implementation.

Current Limitations¶

Compared to the full paper formulation, the following features are not yet implemented:

Paper Feature

Status

AMNA box formulation (Eq. 40-41)

✅ Implemented

Overhang angle constraints (Eq. 42-45)

✅ Implemented

Bridge length constraints (Eq. 46)

✅ Implemented (3D only)

Design space rotation \(\theta_0\)

❌ Not implemented — the build direction is fixed (bottom-to-top)

Design space translation \(y_0\)

❌ Not implemented — the build origin is fixed

Per-component print height \(h_i\)

❌ Not implemented — all layers share a fixed height

KS constraint aggregation (Eq. 47)

✅ Implemented via ks_aggregation in math_utils
